0. Introduction.

Perte de Temps is a solitaire game played with a deck of cards. Its name means ‘‘time waster’’ in French (or at least Google Translate says it does). In this project, you will write a Java program that plays Perte de Temps. Despite its name, this program may still be interesting, because it demonstrates how stacks work. Also, if you have a program that plays Perte de Temps, then you need not waste time playing it yourself.

1. Theory.

A standard deck has 52 cards. Each card has one of thirteen ranks, called ace, two, three, four, five, six, seven, eight, nine, ten, jack, queen, and king. These ranks correspond to the integers 0 through 12, respectively. Each card also has one of four suits, called club, diamond, heart, and spade. These suits correspond to the integers 0 through 3, respectively. A standard deck also has jokers, but they are not used here.
      Perte de Temps is played in the following way. Shuffle the deck, and deal the cards face-down into a row of thirteen piles, with four cards in each pile. This row of piles is called the tableau. Piles are numbered from 0 to 12, so that each pile corresponds to a card rank. Play now proceeds according to the following rules.

  1. Let the pile number p be 0. Draw a card c1 from pile 0.

  2. If pile p is empty, then go to step 7.

  3. Draw a card c2 from pile p.

  4. If c1 and c2 have the same suit, then let p be c1’s rank.

  5. If c1 and c2 have different suits, then let p be c2’s rank.

  6. Throw away c1. Let c1 be c2. Go to step 2.

  7. If all the piles are empty, then the game is won. If some pile is not empty, then the game is lost.

A program that plays Perte de Temps can use arrays and stacks internally. The shuffled deck of cards, before it has been dealt, is represented as an array stack. The tableau is represented as an array of piles. Each pile in the tableau is represented as a linked stack.

Let’s go through the first steps of this game. We start by drawing c₁, the four of clubs with rank 3, from pile 0.
      We draw c₂, the four of hearts with rank 3, from pile 0. The cards c₁ and c₂ have different suits (clubs and hearts), so we’ll draw from pile 3 next. We let c₁ be c₂, the four of hearts.
      We draw c₂, the six of hearts with rank 5, from pile 3. The cards c₁ and c₂ have the same suit (hearts), so we’ll draw from pile 3 again next. We let c₁ be c₂, the six of hearts.
      We draw c₂, the nine of clubs with rank 8, from pile 3. The cards c₁ and c₂ have different suits (hearts and clubs), so we’ll draw from pile 8 next. We let c₁ be c₂, the nine of clubs.
      We draw c₂, the seven of spades with rank 6, from pile 8. The cards c₁ and c₂ have different suits (clubs and spades) so we’ll draw from pile 6 next. We let c₁ be c₂, the seven of spades.
      We continue in this way until we try to draw from pile 12, which is empty. If all the piles were empty, then we’d win the game. Unfortunately, at least one pile is not empty, so we lose. Note that it is very hard to win a game of Perte de Temps! Your program may be correct but never win a game. This is why it’s called ‘‘time waster.’’

3. Implementation.

Your program must use six Java classes called Random, Card, Deck, Pile, Tableau, and PerteDeTemps. Here are descriptions of all these classes and their methods.

The class Random. The Java class library provides a class called Random that implements a random number generator. You must use it to help shuffle the deck. No program can generate truly random numbers, but Random produces numbers that are good enough for our purposes. To use Random, you must put the following line at the start of your program.

import java.util.Random;

Note that java and util are in lower case, Random begins with an upper case letter, and the line ends with a semicolon. The import statement obtains a class from the Java library. The library is divided into packages and subpackages. The package java contains subpackages that are generally useful for writing Java programs. Its subpackage util contains various utility classes, one of which is Random.
      You will need only Random’s constructor and its method nextInt. The constructor call new Random() returns a new instance of the class Random. If r is an instance of Random created in this way, then r.nextInt() returns a randomly generated int. It may be negative, and it may have a large absolute value.
      Your program must use small random int’s, greater than or equal to 0. You can get such an int by executing the Java expression Math.abs(r.nextInt()) % m where m returns an int greater than 0. The class Math is built into Java, and contains public methods for many mathematical functions. The public static method abs implements the absolute value function. The operator % returns the remainder after dividing its left argument by its right argument. As a result, the expression returns a random int greater than or equal to 0, but less than m.
      Hint: Your program must make only one instance of Random. It must use that instance to make all the random int’s that it needs.

The class Card. Each instance of the class Card represents a playing card. Java source code for Card is available in the file Card.java on Canvas. To make your program easier to grade, you must use this source code. You must not extend or modify Card in any way. Here is a short description of Card’s public methods, and what they do.

public Card(int rank, int suit)

Constructor. Initialize a new instance of Card with a given rank and suit. The integer rank must be between 0 and 12. The integer suit must be between 0 and 3.

public int getRank()

Return the rank of this Card, an int between 0 and 12.

public int getSuit()

Return the suit of this Card, an int between 0 and 3.

public String toString()

Return a String that describes this Card, such as "ace (0) of clubs", "two (1) of hearts", "king (12) of diamonds", etc. This String must be used only for printing.

Note that Card’s are immutable objects. No part of a Card can be changed after it is created by its constructor.

The class Deck. An instance of the class Deck represents a deck of Card’s. You must write Deck yourself. It must have the following public methods, and they must work as described here.

public Deck()

(5 points.) Constructor. Make an array containing 52 different Card’s. You must use one or more loops: you will receive no points for this method if you write 52 assignment statements. The order of Card’s within the array does not matter.

public Card deal()

(5 points.) Return the next Card from the array made by the constructor. You need not pick a Card at random, because the Card’s in the array must already be shuffled by the time you call deal. Throw an IllegalStateException if no cards remain to be dealt from the array. This must run in O(1) time.

public void shuffle()

(10 points.) Shuffle the deck of Card’s that is represented by the array made in the constructor. The easiest way is the Durstenfeld-Fisher-Yates algorithm, named after its inventors. It works by exchanging randomly chosen pairs of array elements, and runs in O(n) time for an array of size n. You must use the following pseudocode for this algorithm.

  1. Do steps 2 and 3 for integer values of i, starting from the length of the array minus 1, and ending with 1.

  2. Let j be a random integer between 0 and i.

  3. Exchange the array elements at indexes i and j.

If shuffle is called after any Card’s have been dealt (by the method deal), then it must throw an IllegalStateException. Players are not allowed to shuffle the deck after dealing has started!

Hint: to write Deck, you can use ideas from the array stack class that was discussed in the lectures.

The class Pile. Each instance of the class Pile represents a pile of Card’s. You must write Pile yourself. It must have the following nested classes and methods, which must work as described here.

private class Layer

(5 points.) An instance of this class represents a layer in the Pile. It must have two slots. One must be called card, and it must point to an instance of the class Card. The other must be called next, and it must point to the next Layer in this Pile, or to null. The class Layer must also have a constructor that takes two arguments, called card and next, which determine the values of the two slots.

public Pile()

(2 points.) Constructor. Initialize a new empty Pile of Card’s.

public void add(Card card)

(3 points.) Put card on top of this Pile. This must run in O(1) time.

public Card draw()

(3 points.) Remove a Card from the top of this Pile. Return that Card. Throw an IllegalStateException if there are no Card’s in this Pile. This must run in O(1) time.

public boolean isEmpty()

(2 points.) Return true if this Pile has no cards left in it. Return false otherwise. This must run in O(1) time.

Hint: to write Pile, you can use ideas from the linked stack class that was discussed in the lectures.

The class Tableau. Each instance of the class Tableau represents thirteen Pile’s of Card’s. You must write Tableau yourself. It must have the following methods, which must work as described here.

public Tableau()

(5 points.) Constructor. Make an array with thirteen empty Pile’s in it (see below). Also make a new instance of Deck. Shuffle the Deck. Deal four Card’s from the shuffled Deck into each Pile.

public void play()

(10 points.) Play a game of Perte de Temps. Each time you get a Card from a Pile, you must print:

Got c from pile p.

Here c is a String describing the Card, and p is the number of the Pile you got it from. At the end of the game, if the program won, then print:

We won!

If the program lost, then print:

Pile p is empty. We lost!

Where p is the number of the empty Pile that the program tried to get a Card from, but could not. See the example for details.

Note that play is an exception to our usual rule that methods shouldn’t print things.

The class PerteDeTemps. (0 points.) This is the driver class. It must contain only the main method. You must write PerteDeTemps yourself. Hint: make an instance of Tableau and let it do all the work for main.
